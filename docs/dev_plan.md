# План разработки

# ПРОРАБОТКА ПАНЕЛИ АДМИНИСТРАТОРА

1. Структура и внешний вид панели администратора.

  - Главная страница (сверху вниз):
    - Header с информацией о пользователе: Название сервиса, адрес текущего сервиса.
    - Раздел с выставлением статуса. Компактный. Главная функция - выставить статус "Свободен". Других статусов нет. Время обслуживания убрать.
    - Раздел с управлениями онлайн-записями. Показывается название блока - "Онлайн-записи" и актуальное число записей.
    - Footer с двумя вспомогательными кнопка для быстрой навигации: кнопка "Управлять услугами и ценами" и кнопка "Личный кабинет".

2. Header админ панели.

  - Отображение названия сервиса и его адреса.
  - Если у владельца добавлено несколько точек сервиса, то поле адреса должно представлять собой селектор с выбором точек сервиса по адресу.
  - Слева иконка выпадающего меню, которое содержит разделы в следующей последовательности сврху вниз: "Личный кабинет", "Услуги и цены", "Аналитика".

3. Раздел статуса.

  - Статус можно выставить (включить) только один - "Свободны. Можем принять в ближайшее время".
  - Сделать селектор для включения статуса.
  - Заменить пиктограммы зеленых и желтых кружков для карты на фиолетовый круг (см. брендовые цвета) для обозначения сервиса на карте.
  - Если у сервиса включен статус "Свободен" то к границе фиолетового круга добавить небольшой кружок зеленого цвета (см. брендовые цвета).

4. Раздел онлайн-записей.

  Структура раздела (сверху вниз):
    - кнопка "Добавить онлайн-запись"
     - открывается интерфейс для добавления записи вручную, такая же как если бы клиент записывался.
    - блок для работы с записями по специалистам.
     - отображается ФИО специалиста и кол-во оформленных на него онлайн-записей. При нажатии на специалиста или кол-во записей, открывается основная таблица с соответствующими записями.
    - блок с основной таблицей всех онлайн-записей.
  - Особенность: необходима система гибкой записи. Не просто разместить слоты по 30 мин. в течение рабочего времени сервиса, а дать возможность гибкой настройки (см. "Слоты времени онлайн-записи" в клиентской части)
  - UseCase: Парикмахер должен иметь возможность записать человека в нужное время. Например, к 17:45.
   - Изсохя из того, что известно время для интервальности сетки (самая быстрая услуга в списке активных услуг), то при добавлении такой записи, формируются автоматически свободные слоты на этот день к этому специалисту.
    
5. Footer.
  - Кнопка "Управлять услугами и ценами". Переход в раздел "Услуги и цены".
  - Кнопка "Личный кабинет". Переход в раздел "Личный кабинет".
  
 
## Проработка раздела "Личный кабинет"

  - В раздел "Личный кабинет" вносится вся информация о сервисе.
    - Название, точный адрес с географическими координатами, сфера деятельности (к примеру: автомойка, парикмахерская и пр.).
    - Поле с указанием времени работы по дням недели (эта информация будет задействована при расчете слотов для онлайн-записи и статуса работы).
    - Блок для добавления специалиста: ФИО, фото.
    - Поле для загрузки фотографий сервиса.


## Проработка раздела "Услуги и цены"

  - Раздел "Услуги и цены". Здесь будет вноситься информация о предоставляемых сервисом услугах.
    - Услуги имеют два статуса: активные и снятые в архив. В разделе по UI четко видно раздел активных и ниже то, что добавлено в архив.
    - Поля для заполнения: наименование услуги, цена, время в минутах, за которое сервис оказывает эту услугу, данные специалиста (один или несколько) - выбираются из списка добавленных через блок в личном кабинете.
    - Отдельное окно в поле заполнения услуги по назначению на услугу акции: текст акции, новая цена, срок действия до определенной даты. тумблер активации акции.
    - Если в услуге акция активирована, то визуально перечеркивается старая цена и выделяется новая. На каждую услугу можно добавить несколько акций, но включить можно только одну.
    - Отдельный блок для заполнения информации по специалистам: фото, ФИО.
    - Специалисты должны быть связаны с услугами. Клиент при онлайн-записи сначала выбирает услугу, а затем доступных специалистов, которые оказывают эту услугу.
    - Из функционала: редактирование, удаление, снятие услуги в архив и перенос из архива в активные.


## Проработка раздела "Аналитика"

- Раздел "Аналитика". Здесь отображается информация о клиентах.
  - Метрика, показывающая кол-во просмотров карточки сервиса через клиентское приложение.
  - Метрика, показывающая сколько онлайн-записей получено за текущий месяц. Она же должна показывать кол-во онлайн-записей за предыдущий месяц.
  - Метрика, показывающая, какие услуги и в каком кол-ве были оказаны по онлайн-записи, а также общую сумму полученного дохода по ним исходя из их заявленной цены в разделе "Услуги и цены".
- Раздел "Аналитика" доступен и виден только для мастер пары логин/пароль (для владельца сервиса).
  
  
# ПРОРАБОТКА КЛИЕНТСКОЙ ЧАСТИ.

1. Стурктура и внешний вид клиентской части.
  - Header, слева приветственная полоса "Привет, {имя пользователя}!", а справа скраю "Мое избранное"
   - При нажатии на "Мое избранное" открывается список карточек сервисов, которые клиент добавил себе в избранное.
   - В каждой карточке отображается кратко информация о сервисе: фото, название, адрес.
   - В каждой карточке после информации о сревисе две быстрых кнопки: записаться-онлайн и позвонить.
  - Раздле "Выбор сервиса"
    - Предложение выбрать сервис по категориям
    - Предложение выбрать сервис по услуге
    - "Найти нужный". Поиск сервиса (строка поиска)
  - Карта с выборкой сервисов. Если критерии выбора сервиса не заданы пользователем, то отображаются все сервисы. Если сервисов много на масштаб, то группируются в один кружок с указание кол-ва (как это реализовано на карте сайта etagi.com)
  - При выборе сервиса снизу всплывает карточка сервиса, через которую клиент может записаться-онлайн, позвонить. Карточка отображает название, адрес, фото сервиса, время работы, иконки соц. сетей и кнопки "Услуги и цены" и "Добавить в избранное" (в виде иконки).
    - При нажатии на кнопку "Услуги и цены" открывается список активных услуг с указанием цен.
    
2. Слоты времени онлайн-записи.
 - Сетка слотов имеет особенность гибкости.
 - Клиенту предоставляется три зоны времени в течение дня: "Утро", "День", "Вечер".
 - В каждой зоне указываются свободные интервалы времени.
  - Доступные интервалы высчитываются автоматически исходя из следующих данных:
    - уже существующие записи.
    - график работы.
 - Интервальность сетки слотов равна времени самой быстрой услуги из списка активных услуг.
